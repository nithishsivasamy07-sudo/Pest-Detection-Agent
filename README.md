## рҹ§  Overview

**Pest Detection Agent** is an AI-powered full-stack web application that identifies crop leaf diseases from uploaded photographs. Drop in a `.jpg` or `.png` image вҖ” the system runs a **two-stage AI pipeline**:

1. **Stage 1 вҖ” Vision Classification**: Google Gemini 2.5 Flash analyzes the leaf image and classifies the crop and disease against PlantVillage dataset categories, returning a confidence score.
2. **Stage 2 вҖ” Expert Report Synthesis**: The same Gemini model acts as an agricultural expert вҖ” generating a structured report covering disease overview, visual symptoms, root causes, organic treatments, chemical controls, prevention tips, and direct farmer advice.

All results are persisted in a scan history store and available for review or deletion at any time through the built-in history dashboard.

Two variants ship in this repository:

| Variant | Interface | Backend | Database |
|---|---|---|---|
| рҹҢҗ **Production (Render)** | React + Vite SPA | Node.js / Express | Local JSON (`history.json`) |
| рҹҗҚ **Academic (Docker)** | Same React SPA | Python / Flask + TensorFlow CNN | MongoDB |

## вҳҒпёҸ Deploying to Render

The project ships with a `render.yaml` blueprint вҖ” deployment takes under 5 minutes.

### Steps

1. Push your code to a GitHub repository.
2. Go to [dashboard.render.com](https://dashboard.render.com) вҶ’ **New + вҶ’ Web Service** вҶ’ connect your repository.
3. Render auto-detects `render.yaml`. Confirm:
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `node dist-server/server.js`
   - **Runtime**: Node
4. Under **Environment**, add your secret:

| Key | Value |
|---|---|
| `GEMINI_API_KEY` | `your_key_from_aistudio` |

5. Click **Deploy Web Service**.

The build compiles the React SPA (`vite build вҶ’ dist/`) and the TypeScript server (`tsc -p tsconfig.server.json вҶ’ dist-server/server.js`). The compiled server then serves both static files and API calls from the same Node process вҖ” no separate containers needed.

### Updating the Gemini Model

To switch models without a code change:

1. Render dashboard вҶ’ your service вҶ’ **Environment**
2. Update `GEMINI_MODEL` to any valid model (e.g. `gemini-2.5-pro`)
3. Render auto-redeploys вҖ” done

## рҹ§¬ CNN Model Training

The Python variant ships a full training pipeline for the **PlantVillage dataset**.

### Setup

```bash
# Install Python dependencies
pip install -r requirements.txt

# Structure your dataset:
# dataset/plantvillage/
#   в”ңв”Җв”Җ Tomato Late Blight/
#   в”ңв”Җв”Җ Tomato Healthy/
#   в”ңв”Җв”Җ Potato Early Blight/
#   в””в”Җв”Җ ... (38 classes)

# Run training
python cnn/train.py
```

If no dataset is found, `train.py` automatically **generates synthetic leaf images** using PIL вҖ” drawing leaf shapes, spot patterns, and disease markers per class вҖ” so training always completes even without real images.

### Model Architecture

```
Input (224Г—224Г—3)
    вҶ“
Block 1 вҶ’ Conv2D(32) вҶ’ BN вҶ’ Conv2D(32) вҶ’ MaxPool вҶ’ Dropout(0.25)
    вҶ“
Block 2 вҶ’ Conv2D(64) вҶ’ BN вҶ’ Conv2D(64) вҶ’ MaxPool вҶ’ Dropout(0.25)
    вҶ“
Block 3 вҶ’ Conv2D(128) вҶ’ BN вҶ’ Conv2D(128) вҶ’ MaxPool вҶ’ Dropout(0.40)
    вҶ“
Flatten вҶ’ Dense(512) вҶ’ BN вҶ’ Dropout(0.50)
    вҶ“
Softmax(num_classes)
```

Training uses aggressive data augmentation (rotation Вұ40В°, brightness 0.7вҖ“1.3Г—, vertical+horizontal flip, zoom, shear) with EarlyStopping and ModelCheckpoint. Best model saved to `models/plantvillage_cnn.h5`.

### Standalone Prediction

```bash
python -m cnn.predict path/to/leaf_image.jpg
```

## рҹӣ пёҸ Troubleshooting

| Problem | Solution |
|---|---|
| `GEMINI_API_KEY environment variable is required` | Create a `.env` file in the project root with `GEMINI_API_KEY=your_key` |
| `models/gemini-X.X-flash is not found` | The model was deprecated вҖ” update `GEMINI_MODEL` env var to `gemini-2.5-flash` |
| `Analysis Error: No image file provided` | Ensure `NODE_ENV=production` and the server has started correctly; check Render deploy logs |
| Build fails: `Could not resolve entry module "index.html"` | Run `npm install` вҖ” the `index.html` and `src/main.tsx` entry files must be present |
| `tsx: command not found` on Render | The start command must be `node dist-server/server.js`, not `tsx server.ts` |
| Flask `WinError 10038` on Windows | Expected вҖ” the Flask server disables the watchdog reloader on Windows automatically |
| `ModuleNotFoundError: tensorflow` | Install Python deps: `pip install -r requirements.txt` |
| Docker: `GEMINI_API_KEY not set` | Edit `docker-compose.yml` and set `GEMINI_API_KEY=your_key` under `flask_api.environment` |
